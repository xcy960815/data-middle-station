# 数据中台项目总图

## 系统总图

```text
[Browser]
   |
   v
[Nuxt Pages / Components / Pinia Stores]
   |
   v
[composables/useHttpRequest]
   |
   v
[Nitro API: /server/api/*]
   |
   +--> [AnalyzeService / ChartConfigService]
   |         |
   |         v
   |    [AnalyzeMapper / ChartConfigMapper]
   |         |
   |         v
   |    [MySQL: data_middle_station]
   |    存分析定义、图表配置、定时任务、日志
   |
   +--> [DatabaseService / ChartDataService]
   |         |
   |         v
   |    [DatabaseMapper / ChartDataMapper]
   |         |
   |         v
   |    [MySQL: kanban_data]
   |    存真实业务表 + information_schema 元数据
   |
   +--> [SendEmailService / ScheduledEmailService]
   |         |
   |         +--> [ChartSnapshotService / ECharts SSR SVG]
   |         +--> [SMTP / Nodemailer]
   |         +--> [node-schedule]
   |
   +--> [check-auth middleware]
             |
             +--> [JWT Cookie]
             +--> [Redis]
```

## 分层说明

- `Nuxt Pages / Components / Pinia Stores`
  负责页面展示、交互操作、分析参数组装和前端状态管理。
- `composables/useHttpRequest`
  统一处理前端请求、401 跳转和错误提示。
- `server/api`
  作为前后端边界层，接收请求、校验参数、调用 service。
- `server/service`
  承载业务逻辑，例如分析配置读写、动态查数、邮件发送、定时任务调度。
- `server/mapper`
  直接访问数据库，并通过映射器把数据库字段转换成项目内部对象。
- `data_middle_station`
  系统配置库，存平台自身数据。
- `kanban_data`
  业务数据源，存被分析的真实业务表。
- `Redis`
  用于 token 相关存储和服务端辅助状态。
- `SMTP / node-schedule`
  分别负责邮件发送和定时任务执行。

## 核心业务链路

### 1. 分析页面加载与自动查数

```text
打开 /analyze?id=123
  -> /api/getAnalyze
  -> AnalyzeService.getAnalyze()
  -> analyze + chart_config 回填多个 store

用户选择数据源 / 拖拽字段 / 修改过滤排序
  -> useAnalyzeDataHandler 监听 store 变化
  -> debounce 1s
  -> /api/getAnalyzeData
  -> ChartDataService 动态拼 SQL
  -> ChartDataMapper -> kanban_data
  -> analyzeStore.analyzeData
  -> Chart 组件按 chartType 渲染
```

### 2. 保存分析配置

```text
点击“保存”
  -> 收集 analyze / columns / dimensions / groups / filters / orders / chartConfig
  -> /api/updateAnalyze
  -> AnalyzeService.updateAnalyze()
       -> 创建或更新 chart_config
       -> 更新 analyze 主记录
  -> data_middle_station 持久化完成
```

### 3. 立即发送邮件

```text
点击“邮件” -> 立即发送
  -> /api/sendEmail
  -> SendEmailService.sendMail()
  -> ChartSnapshotService.renderAnalyzeChart(analyzeId)
       -> 读取分析配置
       -> 查询图表数据
       -> ECharts 服务端渲染 SVG
  -> Nodemailer -> SMTP
  -> 记录发送日志
```

### 4. 定时邮件 / 重复任务

```text
前端创建任务
  -> /api/scheduledEmails
  -> ScheduledEmailService.createScheduledEmailTask()
  -> 写入任务表

应用启动
  -> send-emails-regularly 插件加载 pending 任务
  -> node-schedule 注册 job

到达执行时间
  -> executeTaskWithOptions()
  -> processTask()
  -> 复用 SendEmailService.sendMail()
  -> 更新任务状态 + 记录执行日志
```

### 5. 登录与鉴权

```text
/welcome
  -> /api/login
  -> 签发 JWT Cookie
  -> userId / userName / token 写入 Redis

后续访问 /api/*
  -> check-auth 中间件校验 Cookie
  -> 未通过则返回 401
  -> useHttpRequest 捕获后跳回 /welcome
```

## 双库职责

### `data_middle_station`

- `analyze`
  存分析名称、描述、访问次数、`chartConfigId`。
- `chart_config`
  存数据源、字段、维度、过滤、分组、排序、图表公共配置和私有配置。
- `scheduled_email_*`
  存定时任务和执行日志。

### `kanban_data`

- 业务表
  被分析的真实数据表。
- `information_schema`
  为前端数据源选择器提供表名、字段名、字段注释、字段类型。

## 关键代码入口

- 分析初始化：`pages/analyze/useAnalyzeHandler.ts`
- 自动查数：`pages/analyze/useAnalyzeDataHandler.ts`
- 数据源选择：`components/selector/dataSource/index.vue`
- 字段加载：`pages/analyze/components/column/index.vue`
- 保存分析：`pages/analyze/updateAnalyze.ts`
- 动态 SQL 查询：`server/service/chartDataService.ts`
- 系统配置读写：`server/service/analyzeService.ts`
- 图表配置读写：`server/service/chartConfigService.ts`
- 数据库元信息查询：`server/mapper/databaseMapper.ts`
- 邮件发送：`server/service/sendEmailService.ts`
- 图表快照生成：`server/service/chartSnapshotService.ts`
- 定时任务调度：`server/plugins/send-emails-regularly.ts`
- 鉴权中间件：`server/middleware/check-auth.ts`

## 备注

- 这个项目的核心特征不是“前端图表展示”，而是“前端组装分析配置，后端按配置动态查数并支持邮件调度”。
